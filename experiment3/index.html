Experiment 3 - Alternate Worlds

Translated from alternate-dungeon-generator and alternate-overworld-generator on glitch.com

Description

This project is composed of two different world generators with autotiling logics, one generating dungeons with rooms and corridors and another generating the overworld with different biomes created with noise().

Technical

The code generates worlds based on the text editor located next to it, with each symbol representing different tiles and terrains. The user can edit the text editor to make their own world, as well as using the reseed button to randomly generate the worlds.

The dungeon generator uses "_", "-", and ".", where "_" and "-" represents the two different types of lava in the background, and "." represents the floor of the dungeon.

The overworld generator uses "_", "-" and "W", with "_" and "-" representing the two different types of land terrain, highlands and lowlands and "W" being the water.

Reflection

This project was done by myself, and I feel that this project was very successful, with the requirements being fulfilled.

Requirements

  • Prompt: I have two projects, one showing a overworld map with rounded biomes and the other showing a dungeon map with rectangular rooms.
  • Grids: Different random seeds do lead to different grids of codes with visible changes.
  • Autotiling: There are more than two distinct codes used in the grid, and appropriate transitions between the codes are rendered successfully.
  • Editing: The audience is able to edit the text version of the grid to change the scene.
  • Incidentals: The codes are rendered with some additional variation, with each tile having slight variations.
  • Life: The world shows signs of life with animations based on time.
